# 会员相关 UI：完成度记录

> 更新时间：与当前代码库一致；用于稍后调出对照，非实现承诺清单。

## 结论（是否算「会员 UI 做完了」）

**没有。** 顾客端、收银端、管理端的 **MVP 闭环**（注册/登录、余额与流水展示、收银验证储值、后台充值与补录）已具备；**需求 17 中的扩展与合规类 UI**（在线充值、改手机号、注销、导出等）**未做或仅部分**。

---

## 已完成（UI 侧）

| 区域 | 内容 |
|------|------|
| **顾客端** `/:storeSlug/customer/member`（`MemberPortalPage.tsx`） | 登录 / 注册；会员号、余额；修改称呼；修改 PIN；**资料与送餐**：邮编（Eircode）、送餐地址（填写完整邮编后调用与购物车同款的 `/api/geo/customer-eircode` 自动填充，**地址可继续手动修改**）；钱包流水列表；门店入口链接（`StoreFrontPage`） |
| **收银端** `CheckoutFlow.tsx` | 勾选使用储值、手机号 + PIN 验证、展示抵扣与待付、随结账提交 |
| **收银端** `ReceiptPrint.tsx` | 小票展示会员支付与储值抵扣行（若后端传入） |
| **收银端** `ReprintReceipt.tsx` | 退单成功后提示退款渠道分拆；储值退回失败时提示后台补录 |
| **管理端** `admin/members`（`MemberManager.tsx`） | 会员搜索列表；手动充值；Checkout ID **储值退款补录** |
| **管理端** `AdminLayout.tsx` | 侧栏「会员与储值」入口 |
| **i18n** | `zh-CN` / `en-US` 中会员与后台相关文案键（随页面迭代） |

## 未完成或仅部分满足（UI 侧为主；含依赖后端者）

以下条目 **暂不实现**，仅作 backlog，调出本文件即可继续排期。

1. **顾客端在线充值**  
   需求 11「发起/引导充值」：当前 **无** Stripe/扫码付等充值 UI；仅有后台手动充值。若要做，需支付流 + 会员充值接口与页面。

2. **顾客端修改手机号**  
   需求 11：当前 **未提供**；涉及二次验证与合规文案，属「部分满足 / 未做」。

3. **会员注销 / 数据主体请求（UI）**  
   需求 18–19、21：无顾客自助注销；无管理端「导出会员 / 发起注销」等界面（后端流程亦未完整）。

4. **流水与余额的增强展示**  
   如：按类型筛选、关联订单号跳转、导出 PDF/CSV；当前为 **简单列表**。

5. **收银端**  
   - 会员专用结账路径的 **错误与锁定状态**（PIN 锁定）是否全程友好提示：依赖后端错误码，**需逐条验收**。  
   - **全储值（paymentMethod `member`）** 与混合支付的 UI 边界：以 `CheckoutFlow` 与后端为准，若有边角场景需单独列验收项。

6. **顾客扫码点餐与会员**  
   点餐购物车 **未** 与会员登录态打通（例如扫码下单仍用游客，与会员余额无关）；若产品要求「顾客端下单可用储值」，属 **新需求**。

7. **送餐客户档案 UI（与会员相邻能力）**  
   - 收银 `CashierOrder` 送餐：**已** 有按电话拉档案、下拉选档。  
   - 顾客 `CartPage` 送餐下单：**未** 处理 409 必须选 `customerProfileId` 的完整选档 UI（与「会员 UI」相邻，记入以免遗漏）。

8. **无障碍 / 小屏**  
   会员中心与管理端会员页 **未** 专门按 WCAG/小屏做一轮优化。
